Job Application Copilot MVP Spec
==================================

Goal
----
Build a lightweight, human-in-the-loop application copilot that:
1. fetches relevant jobs
2. ranks and queues them
3. prepares application data
4. assists with browser autofill
5. updates the tracker
6. prepares outreach drafts and follow-up tasks

This is not a fully autonomous bot.
Every meaningful step requires user approval.

Primary Outcome
---------------
Help the user move faster on job search without lowering quality.

Success Criteria for MVP
------------------------
The MVP is successful if it can:
- pull matching jobs from target companies and supported ATS sources
- filter out obviously bad-fit roles
- show a ranked "apply now" queue
- launch a guided application flow with prefilled data
- update a spreadsheet after confirmed submission
- prepare personalized outreach drafts for selected people
- create follow-up reminders/tasks in the tracker

Non-Goals
---------
The MVP will not:
- mass apply automatically without approval
- send outreach automatically
- scrape platforms aggressively or violate platform rules
- solve every job portal on day one
- deeply personalize every application using long research
- replace human judgment on final submission

System Philosophy
-----------------
The system should:
- reduce repetitive work
- preserve human control
- be easy to use from both phone and computer
- keep one source of truth
- favor speed and reliability over cleverness

Core User Flow
--------------
Step 1: Job ingestion
- system fetches jobs from approved sources
- deduplicates them
- filters by user preferences
- writes candidates into "New Jobs" queue

Checkpoint 1:
- user approves, rejects, or saves jobs

Step 2: Prioritization
- system scores approved jobs
- system builds first-application queue

Checkpoint 2:
- user confirms queue order

Step 3: Application prep
- system selects best resume version
- system prepares autofill packet
- system flags uncertain questions
- system drafts any free-response suggestions

Checkpoint 3:
- user approves materials

Step 4: Guided application
- system opens browser session
- system fills known fields only
- system pauses for review before submit

Checkpoint 4:
- user reviews filled data

Checkpoint 5:
- user confirms final submission

Step 5: Tracker update
- system updates tracker after confirmed submission
- system sets next action and follow-up date

Checkpoint 6:
- user confirms tracker entry if needed

Step 6: Outreach prep
- system identifies likely people to contact
- system drafts tailored message options

Checkpoint 7:
- user selects recipient, edits draft, and sends manually

Step 7: Follow-up
- system recommends follow-up timing
- system creates follow-up task

Checkpoint 8:
- user approves any follow-up action

MVP Scope
---------
In scope:
- Greenhouse job ingestion
- Lever job ingestion
- Ashby job ingestion
- manual company watchlist
- role classification
- location classification
- match scoring
- spreadsheet sync
- guided browser autofill
- outreach draft generation
- follow-up task generation

Out of scope for v1:
- Workday custom handling beyond minimal/manual
- LinkedIn automation
- automatic email sending
- automatic LinkedIn messaging
- full CRM
- deep company research
- resume rewriting engine
- analytics dashboard beyond simple fields

Target User Preferences
-----------------------
The system should encode the user's preferences:
- target role families:
  - backend engineer
  - infrastructure engineer
  - platform engineer
  - systems/tooling engineer
  - ML infrastructure / inference platform
  - selective full stack if backend-heavy
- avoid:
  - pure QA
  - test-focused roles
  - frontend-heavy roles unless special case
- location:
  - New York based
  - New York office
  - remote
- compensation preference:
  - bias toward roles likely above desired threshold
- company preference:
  - strong technical teams
  - product-driven
  - intentional builders
  - NYC startups and comparable companies
- urgency:
  - newer postings preferred

Functional Requirements
-----------------------

1. Job Source Module
- ingest public jobs from Greenhouse
- ingest public jobs from Lever
- ingest public jobs from Ashby
- optionally ingest manually added company careers URLs
- capture:
  - company
  - role title
  - team if available
  - location
  - source
  - posting URL
  - date posted if available
  - job description text
  - raw ATS metadata

2. Filtering Module
- discard jobs that clearly fail user criteria
- identify remote / NYC / non-match
- identify role family match
- assign reject reason when filtered out

3. Scoring Module
Each role gets a score based on:
- fit score
- interest score or company priority
- freshness
- location match
- referral/outreach potential
- application effort

Initial scoring rubric:
- Fit: 1 to 5
- Company Priority: 1 to 5
- Freshness: 1 to 5
- Location Match: 0 to 3
- Referral Potential: 0 to 2
- Application Effort Penalty: 0 to -3

Formula:
Priority Score =
Fit + Company Priority + Freshness + Location Match + Referral Potential + Effort Penalty

4. Queue Builder
- sort jobs by priority score
- assign action bucket:
  - Apply Now
  - Apply Soon
  - Outreach First
  - Monitor
  - Skip
- allow manual override

5. Resume/Application Packet Module
For each queued role:
- select resume version
- prefill standard fields from profile store
- generate short role-fit summary
- detect likely custom questions
- prepare suggested answers where possible
- flag anything uncertain

6. Browser Assist Module
- open selected job application in browser
- fill standard fields:
  - name
  - email
  - phone
  - location
  - LinkedIn
  - GitHub
  - portfolio if available
  - work authorization
  - resume upload
  - common demographic/work eligibility questions if explicitly configured
- stop on:
  - ambiguous questions
  - custom long-form answers
  - broken selectors
  - suspicious or low-confidence actions
- require final approval before submit

7. Tracker Sync Module
After confirmed submission:
- create or update tracker row
- populate:
  - company
  - role
  - title bucket
  - location
  - source
  - date posted
  - priority score
  - resume version used
  - date applied
  - contact name
  - contact type
  - outreach goal
  - outreach sent
  - response
  - stage
  - next action
  - last touched
  - follow-up due
  - application URL
  - notes

8. Outreach Module
- identify 1 to 3 likely contacts:
  - recruiter
  - hiring manager / team lead
  - engineer or alum
- draft message options by recipient type:
  - recruiter
  - hiring manager
  - engineer
  - warm contact
  - founder/early leader
- include:
  - why this company / role
  - why user fits
  - what ask is being made
- never send automatically

9. Follow-up Module
Default timing rules:
- cold application:
  - outreach same day or next day
  - one follow-up after 5 to 7 business days
- warm contact:
  - contact first
  - wait 1 to 3 days
  - apply anyway if no response
- high-priority startup:
  - apply and reach out same day

Ranking Logic Details
---------------------
Fit Score (1 to 5)
- 5 = very aligned backend/infrastructure/platform/ML infra role
- 4 = generally aligned software/backend role
- 3 = adjacent role with some relevance
- 2 = weak fit
- 1 = poor fit

Company Priority (1 to 5)
- 5 = top target company
- 4 = strong target
- 3 = solid company, moderate excitement
- 2 = backup
- 1 = low priority

Freshness (1 to 5)
- 5 = posted in last 3 days
- 4 = posted in last 7 days
- 3 = posted in last 14 days
- 2 = posted in last 21 days
- 1 = older or unknown

Location Score (0 to 3)
- 3 = NYC based or NYC office role
- 2 = remote
- 1 = plausible hybrid with New York relevance
- 0 = non-match

Referral Potential (0 to 2)
- 2 = warm path exists
- 1 = cold but reachable
- 0 = unclear

Effort Penalty (0 to -3)
- 0 = easy application
- -1 = moderate tailoring
- -2 = high-friction portal or moderate custom questions
- -3 = very high effort

Outreach Rules
--------------
Default rules:
- do not message more than 3 people per company per role
- usually message 1 recruiter + 1 manager/engineer
- follow up once unless there is engagement
- stop after second touch unless a reply exists

Message types supported:
- recruiter intro after application
- hiring manager direct note
- engineer insight request
- warm contact advice request
- founder/startup note

Automation Boundaries
---------------------
The system may:
- fetch
- classify
- rank
- draft
- prefill
- log

The system may not without approval:
- submit an application
- send a message
- alter final free-response answers
- apply to demographic/authorization questions unless explicitly configured
- mark application successful without confirmation
- spam multiple contacts

Error Handling
--------------
If job ingestion fails:
- log source error
- keep prior data
- notify user in dashboard

If browser fill fails:
- stop immediately
- highlight failed field
- allow manual continuation

If tracker sync fails:
- queue pending update
- notify user
- retry manually or automatically later

If contact discovery is weak:
- fall back to manual contact entry

Security / Privacy
------------------
- store personal profile data securely
- keep resume and personal answers local or in trusted storage
- do not log sensitive application answers unnecessarily
- avoid prohibited automation on platforms with clear restrictions
- make all approval points explicit

MVP Acceptance Tests
--------------------
1. Job ingestion test
- system pulls jobs from at least 3 sources
- system deduplicates repeat postings
- system writes matching jobs into queue

2. Filtering test
- obvious non-matching roles are filtered out
- NYC/remote preference is correctly reflected

3. Queue test
- system ranks jobs in sensible order
- user can approve/reject/reorder

4. Application prep test
- system selects a resume version
- system generates autofill packet
- system flags uncertain fields

5. Browser assist test
- system fills a supported application form
- system pauses before submit
- user can manually edit everything

6. Tracker sync test
- confirmed application updates spreadsheet correctly

7. Outreach draft test
- system generates at least 2 recipient-specific drafts for a chosen application

8. Follow-up test
- system creates follow-up due date according to rules

MVP Deliverables
----------------
At MVP completion, the system should provide:
- a running job fetch pipeline
- a live queue of ranked roles
- a live spreadsheet tracker
- guided autofill for supported applications
- structured outreach drafts
- follow-up recommendations
